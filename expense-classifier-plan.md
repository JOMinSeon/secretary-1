# 지출 자동 분류기 (AI Expense Classifier) 개발 계획서

## 1. 개요
홈택스/카드사 연동 기반의 결제 내역을 AI(Gemini)를 통해 자동으로 분류하고, 부가세 공제 여부 및 절세 팁을 제공하는 서비스입니다. 프리랜서 및 개인사업자가 복잡한 세무 지식 없이도 지출을 관리할 수 있도록 돕습니다.

## 2. 프론트엔드 개발 계획

### 2.1 UI/UX 구성
- **대시보드 상단**: 총 지출, 공제 가능 금액, 예상 환급액 요약 카드
- **차트 영역**: 카테고리별 지출 비중 시각화 (Recharts 사용)
- **액션 영역**: CSV/Excel 파일 업로드 영역 및 'AI 자동 분류 시작' 버튼
- **테이블 영역**: 지출 상세 내역 리스트 (shadcn/ui Table)
    - 컬럼: 일자, 가맹점명, 금액, 카테고리(AI), 공제여부(AI), 절세 팁

### 2.2 주요 기능 구현
- **파일 파싱**: `xlsx` 라이브러리를 사용하여 업로드된 엑셀/CSV 데이터를 JSON으로 변환
- **데이터 상태 관리**: `zustand` 또는 `react-query`를 사용하여 지출 내역 관리
- **AI 트리거**: 분류 버튼 클릭 시 백엔드 API 호출 및 실시간 상태 업데이트 (pending -> completed)

## 3. 백엔드 개발 계획

### 3.1 API Route (Next.js)
- **Endpoint**: `/api/classify` (POST)
- **로직**:
    1. 클라이언트로부터 지출 내역(가맹점명, 금액 등) 수신
    2. Gemini AI에 프롬프트 전달 (가맹점명 기반 카테고리 추론 및 세법 기준 공제 판별)
    3. 결과값(JSON)을 수신하여 DB 업데이트 또는 반환

### 3.2 데이터베이스 (Supabase)
- **Table**: `receipts` (기존 테이블 확장)
    - 추가 컬럼: `tax_tip` (text), `status` (text)
- **RLS**: 사용자별 데이터 접근 제한 정책 적용

## 4. AI 프롬프트 전략
- **역할 부여**: "당신은 한국 세무 전문가이자 AI 지출 분류기입니다."
- **입력 데이터**: 가맹점명(예: '스타벅스'), 금액, 날짜
- **추론 로직**: 
    - 업종 매핑: 스타벅스 -> 카페 -> 복리후생비(식대) 또는 접대비
    - 공제 판별: 사업 관련성 기반 공제 여부 판단 기준 제공
- **출력 형식**: 구조화된 JSON (category, is_deductible, tax_tip)

## 5. 단계별 일정 (MVP)
1. **1일차**: DB 스키마 수정 및 백엔드 API(Gemini 연동) 개발
2. **2일차**: 파일 업로드 및 파싱 로직 구현
3. **3일차**: 대시보드 UI 연동 및 차트 시각화
4. **4일차**: 최종 테스트 및 버그 수정
